# Studio Architecture

## Layers

- Control Center: orchestration and governance
- Studio Core: design authoring and control
- Registry: identity and asset authority
- Runtime: validation and enforcement
- UI: visible control surface
- Marketplace: reusable assets and extensions

## Studio Role

The studio is the missing layer between governance and public creation.

It must support:
- themes
- palettes
- typography
- buttons
- templates
- components
- style packs
- asset IDs
- publishing flows

## Required Rule

Nothing in the studio should be hardcoded when it can be governed by tokens, registry entries, or authored style rules.
