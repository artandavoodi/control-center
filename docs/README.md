# Documentation

This directory holds the written continuity layer for Control Center.

## Purpose

Documentation here should preserve:

- system intent
- architectural boundaries
- current status
- engine definitions
- workflow rules
- implementation history

## What Belongs Here

- handovers
- architecture notes
- state reports
- implementation summaries
- reference maps
- continuation instructions

## Required Discipline

- keep documentation aligned with live repository structure
- do not invent alternate ownership paths
- do not drift from the active architecture
- keep language direct and operational

## Recommended Files

- `System Overview.md`
- `Architecture Map.md`
- `Current Status.md`
- `Change Log.md`
- `Implementation Notes.md`

## Current State

This layer is intentionally minimal at the start and should expand as the system becomes more formalized.
